# ふりかえりアプリ 回帰チェック

## 使い方

- 本番反映前に、最低限このチェックだけ通す。
- まず `開発用マスター` で確認し、必要なときだけ `deploy:core` または `deploy:full` を行う。
- 異常があれば本番反映しない。

## A. 画面起動

- [ ] 先生画面URLが開く
- [ ] 児童画面URLが開く
- [ ] 教師画面の build 表示が止まらない
- [ ] 教師画面で単元一覧が表示される
- [ ] 児童画面で授業待ち / 授業中の表示が正常

## B. 単元と授業開始

- [ ] 単元を追加できる
- [ ] 単元を更新できる
- [ ] 単元を削除できる
- [ ] 単元追加 / 削除後に一覧表示が更新される
- [ ] 授業スタートタブで単元を選べる
- [ ] 授業を開始すると授業中表示になる
- [ ] 授業終了できる

## C. 児童記録

- [ ] 児童画面で入力できる
- [ ] 自動保存される
- [ ] 提出できる
- [ ] 提出後、教師画面に反映される
- [ ] 既存記録が消えていない

## D. 教師フィードバック

- [ ] AI未設定でも授業状況タブが開く
- [ ] AI未設定でも手入力の下書き保存ができる
- [ ] AI未設定でも返却できる
- [ ] AI未設定でも手動メダル付与ができる
- [ ] AI設定済みならAI下書き生成ができる

## E. 互換と既存利用者保護

- [ ] 既存の先生URLが引き続き開く
- [ ] 既存の児童URLが引き続き開く
- [ ] 既存シートの単元が読める
- [ ] 既存シートの記録が読める
- [ ] 旧データ形式でも例外停止しない

## F. 配布・導入

- [ ] 導入管理ページが開く
- [ ] 配布用テンプレートが開く
- [ ] 配布用テンプレートからコピーできる
- [ ] 初期設定メニューが出る
- [ ] `つぎへ` シートの案内が開ける

## G. デプロイ後の記録

- [ ] 本体 version を記録した
- [ ] 配布テンプレート version を記録した
- [ ] 導入管理 version を記録した
- [ ] provision version を記録した
- [ ] 必要なら `HANDOFF.md` を更新した

## メモ

- Chrome で開けないときは `シークレット` と `Edge` でも確認する。
- 既存利用者を壊しそうな変更では、`実運用テナント` でも最低1回確認する。
