# プロジェクト記憶

## 目的
小学校教師が、授業中の児童のふりかえりを集約し、AIコメントや教師の見取りに活用するWebアプリ。

## 想定ユーザー
- 教師
- 小学校3年生程度の児童
- 40人学級でも使えることを想定

## 技術
- Google Apps Script
- Googleスプレッドシート
- HTML / CSS / JavaScript
- Gemini API
- Cloudflare Pages 側のWeb画面も使用する可能性あり

## 大事な思想
- 画面はできるだけ中央更新できる形にする。
- データは先生ごとのスプレッドシートに残す。
- 授業中に重くならないことを優先する。
- 児童操作はできるだけ単純にする。
- 教師の操作は1〜2クリックで済むようにする。

## 現在の実装方針
- 主配布導線は Web 版だが、GAS は保存本体・API・導入管理の母体として残す。
- `src/index.html` / `src/teacher.html` は現場の主入口というより、GAS互換UI兼 portable 生成元として扱う。
- 提出保存と AI 処理は分離し、提出時はまず保存、AI はキュー処理で行う。
- 単元設定は授業時点の項目を保持できるようにする。

## よくある課題
- iPadでの表示崩れ
- 40人同時入力時の競合
- GAS/API の遅延
- 出席番号と児童名変換の誤作動
- AIコメント生成の遅さ

## 運用メモ
- 変わりやすい URL、version、直近変更、未解決事項は `HANDOFF.md` に書く。
- ファイルやディレクトリの役割は `FILEMAP.md` に書く。
