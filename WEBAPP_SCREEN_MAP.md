# GAS版 -> Web版 画面マップ

## 方針

- `残す`
  - 現場で通った導線
  - 先生が迷いにくかった並び
  - 児童が触りやすかった画面分割
- `捨てる`
  - GAS配布や認可のためだけに増えた画面
  - スプレッドシート前提の操作
- `組み替える`
  - 情報は同じだが、Web版で構造を簡潔にする画面

## 児童画面

### そのまま採用

#### 1. 出席番号選択

- 現状:
  - `screen-select`
  - 大きい番号ボタン
  - 番号の再選択導線あり
- Web版:
  - そのまま採用
  - 最初の顔として維持

#### 2. 時間選択

- 現状:
  - `screen-period`
  - 単元と時間目を見て選ぶ
- Web版:
  - そのまま採用
  - ただし `授業中なら自動選択` の余地は残す

#### 3. 入力画面

- 現状:
  - 左にみんなの記録
  - 右に自分の入力
  - 提出とAIコメント
- Web版:
  - 主構成として採用
  - iPad前提で縦積み / 横並び切替

#### 4. これまで表示

- 現状:
  - `これまで` タブ
- Web版:
  - タブ構成を維持
  - 児童の自己比較導線として残す

### 組み替える

#### 5. みんなの記録タイムライン

- 現状:
  - 同画面内に表示
- Web版:
  - メイン画面の補助パネルとして残す
  - 小画面では折りたたみへ

#### 6. AI到着バナー

- 現状:
  - 到着時にコメントへジャンプ
- Web版:
  - 残す
  - ただし通知表現は軽く整理

## 教師画面

### そのまま採用

#### 1. 授業スタート

- 現状:
  - 最重要タブ
  - 単元選択
  - 授業開始
  - 単元設定へのジャンプ
- Web版:
  - ホーム画面の主役として採用
  - 最初に開く画面にする

#### 2. 単元設定

- 現状:
  - 単元一覧
  - 新規追加
  - 項目設定
- Web版:
  - ほぼそのまま採用
  - 教科デフォルトは後回し

#### 3. 名簿

- 現状:
  - 出席番号と名前
- Web版:
  - そのまま採用
  - CSV一括取込は後で追加

#### 4. 記録 / ポートフォリオ / 評定

- 現状:
  - `記録・所見・評定`
  - 単元記録
  - ポートフォリオ
  - 評定
- Web版:
  - `記録` と `返却` と `ポートフォリオ` を先に移植
  - `評定` は後段

### 組み替える

#### 5. 使い方

- 現状:
  - 配布やAPIキー説明が多い
- Web版:
  - 初回セットアップとヘルプへ再構成
  - 配布説明は不要化

#### 6. 設定・AIプロンプト

- 現状:
  - AI設定
  - 更新
- Web版:
  - `設定` に統合
  - `更新` タブは不要

#### 7. AIログ

- 現状:
  - 保守向けログUI
- Web版:
  - 管理者用に隠す
  - 一般教師UIからは外す

### 捨てる

#### 8. GASセットアップ導線

- 例:
  - デプロイURL反映
  - 更新認可
  - 配布テンプレート前提の導線
- Web版:
  - 全廃

#### 9. Apps Script API 前提の更新UI

- 例:
  - 自己更新
  - 版戻し
- Web版:
  - SaaS型の通常デプロイへ置換

## Web版の新しい画面構成

### 教師

1. ホーム
2. 授業スタート
3. 授業中モニタ
4. 単元設定
5. 名簿
6. 記録一覧
7. 児童別ポートフォリオ
8. 返却
9. 設定

### 児童

1. 出席番号
2. 時間選択
3. 入力
4. 返却確認
5. これまで

## 優先順位

### 最優先で移す

- 児童: 出席番号 -> 時間 -> 入力 -> 提出
- 教師: 授業開始 -> 提出確認 -> 返却
- 教師: 単元設定
- 教師: 名簿

### 後で移す

- ポートフォリオ
- 単元レポート
- AI下書き
- 評定

### 初期MVPでは外す

- 保守ログ
- 更新機能
- GAS配布系ヘルプ

## 実装メモ

- 児童画面の文言は GAS版に寄せる
- 教師画面の最初の画面は `授業スタート`
- `単元設定へ` のジャンプ導線は残す
- 児童と教師の両方で `迷ったら戻れる` ボタンを置く
